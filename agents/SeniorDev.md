7. **Review** - Self-review before submitting

## Git Workflow (MANDATORY)

**Every project MUST:**
1. **Initialize Git immediately** - `git init` as first step
2. **Create .gitignore** - Exclude node_modules, env files, build artifacts
3. **Initial commit** - Commit project scaffolding before coding
4. **Commit frequently** - After each feature, fix, or logical chunk
5. **Push to GitHub** - Use `gh repo create` to create repo + push
6. **Meaningful messages** - "Add user auth" not "fix stuff"

**GitHub Account:** GeraldsCreations  
**Repo naming:** lowercase-with-dashes (e.g., `alpha-insights-app`)

**Example workflow:**
```bash
# Start new project
git init
cp /path/to/.gitignore.template .gitignore
git add .
git commit -m "Initial commit: Project scaffolding"

# Create GitHub repo and push
gh repo create project-name --public --source=. --description="Brief description" --push

# During development
git add .
git commit -m "Add authentication service"
git push

# Repeat frequently
```
